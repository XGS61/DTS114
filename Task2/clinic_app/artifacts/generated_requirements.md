# Requirements Summary

## Business Problem
Clinic staff need a safe appointment administration prototype that separates patient booking from doctor/admin review, lets patients choose a date, doctor, and available slot, keeps one shared SQLite workflow, and records testing, CI/CD, Docker, deployment, and AI generation evidence.

## Scope
Appointment administration prototype only. No diagnosis, treatment advice, prescriptions, or real patient records.

## Actors
- Patient: can register, book, cancel (own), reschedule (pending), view own appointments.
- Doctor: can review own queue, manage profile, view day-board, audit events.
- Admin: can manage all queues, create doctor accounts/profiles, review any appointment.
- Clinic System: enforces safety, scheduling rules, authentication, persistence.

## Functional Requirements
- Role-based authentication (register, login, logout).
- Patient selects date, doctor, slot; requests appointment.
- Doctor/Admin review with required notes; statuses: Pending Review, Confirmed, Cancelled.
- Patient cancels own request; reschedule pending requests.
- Doctor manages own profile; Admin manages doctor accounts.
- Scheduling: future weekdays, doctor shifts, 30-min slots, capacity, allowed types.
- Audit events for all changes.
- Metadata endpoint for requirements.
- Safety validation rejects diagnosis/treatment/prescription wording.

## Non-Functional Requirements
- SQLite persistence with automatic schema.
- Hashed demo credentials.
- English-only, ASCII.
- Pytest tests, Docker, CI/CD, Render deployment.
- Evidence of AI generation (DeepSeek, APIFREE).

## AI-DLC Traceability
- Requirements, user stories, validation checklist, UML diagrams generated by DeepSeek.
- Image and doctor photos generated by APIFREE.
- Repository structure, CI/CD, and deployment configured.
